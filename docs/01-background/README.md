# 01-background

## 项目背景
为医疗机构提供一套轻量化的当天挂号系统，覆盖患者移动端自助挂号和后台管理端的订单/就诊人管理。系统采用 SQLite 单机部署，适合中小型诊所、社区医院等无需复杂运维的场景。

## 目标用户
1. **患者 / 就诊人**：通过移动端（H5）查看当天号源、添加就诊人信息、完成挂号。
2. **医疗机构管理人员**：通过管理端（PC Web）查看挂号订单、管理就诊人档案、处理退号/改号等业务。

## 核心痛点
1. 传统挂号窗口排队时间长，患者体验差，需要线上自助渠道。
2. 就诊人信息每次重复填写，缺乏统一的档案管理。
3. 后台管理人员缺乏实时可视化的挂号订单看板，难以掌握当天号源使用情况。
4. 中小型医疗机构缺乏轻量、低运维成本的挂号系统。

## 目标终端
- **Web PC**：管理端（后台/仪表盘），面向医疗机构管理人员
- **H5**：移动端，面向患者/就诊人

## 竞品分析（可选）
| 竞品 | 优点 | 缺点 |
|------|------|------|
| 医院官方 App | 功能全面、号源多 | 开发成本高、需要独立运维、不适合中小诊所 |
| 第三方挂号平台 | 流量大 | 抽成高、数据不在本地、无法自定义流程 |
