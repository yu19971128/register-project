# 模块：当天挂号

## 职责
- 提供移动端（H5）当天挂号的完整流程：浏览当天可用号源、选择就诊人、确认挂号信息、提交挂号、生成挂号凭证。
- **后端核心**：接收挂号请求，校验号源余量和就诊人有效性，原子性扣减号源余量，创建挂号订单记录，返回唯一挂号凭证（订单号 + 二维码/条形码）。
- **挂号凭证**：生成包含订单号、就诊信息、时间的凭证，供患者到院出示。

## 边界
- **不做就诊人管理**：就诊人的增删改查属于「就诊人管理」模块，本模块只读取已有就诊人信息。
- **不做号源配置**：号源的创建、编辑、删除属于「号源管理」模块，本模块只查询可用号源并调用扣减接口。
- **不做订单查询/退号/改号**：挂号订单的列表查询、退号、改号、状态流转属于「挂号订单管理」模块。
- **不处理支付**：系统暂不支持在线支付，挂号即视为预约成功。

## 依赖模块
- **就诊人管理**：获取当前访客关联的就诊人列表，校验就诊人有效性
- **号源管理**：查询当天可用号源，调用扣减接口锁定号源
