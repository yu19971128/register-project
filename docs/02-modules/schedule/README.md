# 模块：号源管理

## 职责
- 提供当天号源的全生命周期管理：创建、查询、更新、删除。
- **管理端（PC）**：管理员可配置当天号源，包括科室、医生、出诊时间段、总号数。支持按日期查看和管理号源。
- **余量管理**：挂号时自动扣减号源余量，退号/改号时回滚余量。确保号数不超卖。
- **对外查询**：为「当天挂号」模块提供当天可用号源查询接口（按科室/医生/时间段筛选）。

## 边界
- **不做挂号逻辑**：患者选择号源并提交挂号的行为属于「当天挂号」模块，本模块只提供号源数据和余量操作接口。
- **不做就诊人管理**：就诊人档案属于「就诊人管理」模块。
- **不做订单状态流转**：挂号订单的创建、查询、退号/改号属于「挂号订单管理」模块。
- **不处理支付**：系统不涉及在线支付，挂号成功后即生成订单。
- **业务聚焦当天号源**：当前核心场景为当天挂号，号源 date 字段默认当天；技术层面支持任意日期，为后续扩展预留空间。管理端创建号源时 date 默认为当天，但可手动指定其他日期。

## 依赖模块
