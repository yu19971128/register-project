# 挂号订单管理 — 前端设计

## 用途
提供挂号订单的查询和管理界面，覆盖管理端（全部订单操作）和移动端（患者自助查询/退号）。

---

## 页面清单

| 页面 | 端 | 路由 |
|------|-----|------|
| 挂号记录列表 | H5 | /h5/orders |
| 订单详情（H5） | H5 | /h5/orders/:id |
| 挂号订单列表 | PC | /orders |
| 订单详情（PC） | PC | /orders/:id |

---

## 页面 1：挂号记录列表（H5）

### 布局结构

```
┌─────────────────────────────┐
│  ← 挂号记录                  │  ← Header (44px)
├─────────────────────────────┤
│  [全部] [待就诊] [已就诊] [已退号]│  ← 状态筛选标签
├─────────────────────────────┤
│  ┌─────────────────────┐   │
│  │ 🩺 内科 · 王医生      │   │
│  │ 2026-04-29 09:00-10:00│  │
│  │ 就诊人：张三          │   │  ← 订单卡片
│  │ 状态：🟢 待就诊        │   │
│  └─────────────────────┘   │
│  ┌─────────────────────┐   │
│  │ 🩺 外科 · 李医生      │   │
│  │ 2026-04-28 10:00-11:00│  │
│  │ 就诊人：李四          │   │

│  └─────────────────────┘   │
│         ···                 │
└─────────────────────────────┘
```

### 说明
- 顶部返回箭头 + 标题
- 状态标签横向筛选：全部/待就诊/已就诊/已退号
- 订单卡片展示科室、医生、时间、就诊人、状态
- 点击卡片进入订单详情
